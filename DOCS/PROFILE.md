# Профиль игрока

## Обязательные поля при регистрации (первый вход)

| Поле | Требование |
|------|------------|
| Имя / ник | Обязательно, **уникально** в системе |
| Email | Обязательно |
| Возраст **или** год рождения | Обязательно (один из вариантов — конкретизация в ТЗ на формы) |
| Пол | Обязательно; для **агрегированной статистики**, на **рейтинг не влияет** |
| Клуб | Опционально; рекомендуется для вовлечения **локального комьюнити** |
| Регион / район города | Опционально |

## Дополнительные поля (онбординг, можно пропустить)

| Поле | Описание |
|------|----------|
| Рост, вес | Опционально |
| **Эквип** | Опционально; **свободный текст** (или несколько строк): что использует игрок — ракетка, струны, обувь, одежда и т.д. **Без обязательного справочника брендов**; пользователь сам указывает формулировки, чтобы не ограничивать себя. |
| Фото / аватар | Опционально |
| Текст «О себе» | Уровень, стиль игры, достижения вне портала, при желании — самозаявка в духе NTRP и любая другая текстовая информация; **отдельного поля под NTRP нет**. |

### Статус готовности (задаёт владелец профиля)

- **«Ищу вызов»** — игрок **отображается** в выдаче/списке для вызовов как дополнительный маркер готовности сыграть.
- **«Временно не играю»** — на вызовы влияет по правилам продукта (например, исключение из активной выдачи или пометка недоступности); **профиль** для просмотра остаётся доступен.

Детализация UX и правил фильтрации — в ТЗ на вызовы и каталог игроков.

## Системные блоки профиля (не редактируются вручную)

Отображаются на странице профиля **владельцу и другим пользователям** (только просмотр), источник данных — матчи и сезон портала:

- **Матчи** — сводка и/или история сыгранных матчей на площадке (формат списка и глубина — в ТЗ на матчи и UI).
- **Win / Loss** — визуализация (графики/диаграммы) побед и поражений за выбранные периоды.
- **Текущий рейтинг** — актуальное значение **сезонного рейтинга портала**.
- **Место в сезоне** — позиция в **текущей сезонной таблице** (в рамках того разреза, в котором участвует игрок).

**Достижения** — полный список полученных ачивок на профиле; правила выдачи и каталог — **отдельное ТЗ**.

## Видимость для других пользователей

Редактируемые поля (анкета): **кроме статуса готовности** для других пользователей **видны только просмотр** (readonly). **Статус** используется в **контексте вызовов** (списки, подбор); на **карточке чужого профиля** не отображается (или иное правило — зафиксировать в UI-ТЗ).

**Системные блоки** (матчи, Win/Loss, рейтинг, место в сезоне, достижения) — всегда **только данные портала**, без ручного редактирования; видимость для чужих профилей как правило **полная**, если отдельным документом не введены исключения.

Редактировать поля анкеты может **только владелец** профиля (в рамках правил модерации и валидаторов).
