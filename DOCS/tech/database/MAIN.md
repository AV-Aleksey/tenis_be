# Database Stack

- DBMS: PostgreSQL
- ORM: Prisma

## Принятое решение

Для backend-приложения используем PostgreSQL как основную реляционную БД.  
В качестве ORM используем Prisma: низкий порог входа, понятный синтаксис и качественная официальная документация.

## Основные требования к хранению данных

- Часовой пояс и даты: все даты и время храним в UTC, используем `timestamptz`.
- Индексы: добавляем индексы на часто используемые фильтры, сортировки, внешние ключи и поля поиска.
- Аудит: для ключевых таблиц обязательны поля `created_at` и `updated_at`.
- Безопасность: чувствительные данные (например, email, телефон) не выводим в логи в открытом виде.
