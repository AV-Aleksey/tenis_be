# Backend Common (MVP)

## Модули

### 1) auth-gateway
Единая точка входа: JWT auth (access), проверка прав, маршрутизация запросов в доменные модули.

### 2) profile
Профиль игрока: данные игрока, `slug`, рейтинг и счетчики игр.

### 3) filestorage
Прием и хранение файлов (аватар): валидация mime/size, сохранение в `uploads/`, возврат пути для profile.

### 4) challenge
Полный цикл вызова/матча: создание вызова, принятие/отклонение, подтверждение результата, спор, финализация, пересчет рейтинга.

### 5) audit
Журналирование критичных действий и изменений состояния (кто/что/когда/с чем).

---

## Связи модулей

- Все внешние запросы идут через `auth-gateway`.
- `auth-gateway -> profile`: синхронный вызов `profile.create(...)` при регистрации.
- `auth-gateway -> challenge`: операции по вызовам/матчам.
- `profile -> filestorage`: загрузка/обновление аватара.
- `challenge -> profile`: чтение рейтинга/игроков, запись нового рейтинга и total_games после финализации.

Ограничения:
- `filestorage` не знает про бизнес-логику challenge/profile, только про файлы.
- `audit` не влияет на бизнес-решения, только принимает события.

---

## MVP-флоу

1. Игрок регистрируется через `auth-gateway` и получает `access`.
2. `auth-gateway` синхронно вызывает `profile.create(...)`.
3. `profile` создает профиль и связывает его с `auth_user_id`.
4. Игрок задает `slug` в `profile`.
5. Игрок загружает аватар: `profile -> filestorage`, путь сохраняется в `profile`.
6. Игрок создает вызов: `auth-gateway -> challenge`.
7. Соперник принимает/отклоняет вызов в `challenge`.
8. После игры стороны подтверждают результат в `challenge`.
9. При расхождении подтверждений `challenge` переводит матч в спор.
10. При финализации `challenge` обновляет рейтинг и total_games через `profile`.

---

## Технические правила

- API: REST `api/v1`.
- Auth: JWT access 15m.
- DTO валидация: `ValidationPipe` (`whitelist`, `forbidNonWhitelisted`, `transform`).
- Файлы: только `jpeg/png/webp`, до `5MB`.
- Завершенный матч для статистики: только финальный статус `completed`.
- Во внешних API и UI использовать `slug`, внутренние `id` не отдавать.
