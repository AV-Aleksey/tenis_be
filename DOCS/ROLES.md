# Роли и доступ

Целевая платформа: **TG-бот**.

В системе используется **одна роль: Admin**. Дополнительные роли RBAC (Moderator/User) не применяются.

## Пользователь (игрок)

- Пользователь бота существует как доменная сущность (игрок), но **не как роль доступа**.
- Доступ к обычным пользовательским сценариям определяется логикой продукта, без отдельного слоя RBAC для `User`.
- Административные действия и интерфейс правок доступны только роли **Admin**.

## Admin

- Имеет полный доступ к рабочим сценариям бота.
- Выполняет административные действия: правки данных, управление сезонами, изменение глобальных настроек.
- Работает через отдельный административный интерфейс (детализация будет вынесена в отдельное ТЗ).

## Аудит действий Admin

- Любое существенное действие администратора фиксируется в журнале: кто выполнил действие, когда, тип операции, затронутые сущности (матч, пользователь, сезон, настройка), при наличии — основание/комментарий.
- Минимум покрытия: ручная правка результата, блокировка/разблокировка, изменения сезона, изменения глобальных настроек.
- Просмотр журнала доступен роли **Admin**.
- Технический формат хранения, сроки ретенции и экспорт — в ТЗ на бэкенд и админ-интерфейс.
